## 2. `alphav()` NotImplemented 크래시

**증상:** `Not implemented — SRKGas::alphav()` → 솔버 abort  
**원인:** OFv8에서 `return 0`이었던 `alphav()`를 OF-13 포팅 시 `NotImplemented`로 남겨둠. OF-13의 chemistry jacobian(`dkrdT`)이 `alphav()`를 호출함.

**수정 (`SRKGasI.H`):**
```cpp
// 수치 미분으로 구현
const scalar dT = min(T*1e-6, 1e-4);
const scalar rhoT = rho(p, T);
const scalar rhoTp = rho(p, T + dT);
const scalar rhoTm = rho(p, T - dT);
return -(rhoTp - rhoTm)/(2*dT*rhoT);
```

## 3. `sv()` NotImplemented

**증상:** 잠재적 크래시 위험  
**수정 (`SRKGasI.H`):**
```cpp
return sp(p, T) - CpMCv(p, T)/max(T, small);
```

## 4. dynamicCode 런타임 컴파일 시 헤더 경로 문제

**증상:** `rfSpecie.H: No such file or directory` (dynamicCode의 chemistry model 런타임 컴파일 실패)  
**원인:** OF-13이 chemistry/thermo를 런타임에 컴파일할 때 `/opt/openfoam13/etc/codeTemplates/dynamicCode/` 템플릿의 `codeOptions`만 참조. RGP-13 헤더 경로가 없음.

**수정:**
- `codeTemplates/dynamicCode/` 디렉토리 생성, 원본 템플릿 복사 후 RGP-13 include 경로 추가
- `basicChemistryModel`, `fluidMulticomponentThermo`, `psiThermo`, `fluidThermo`, `psiuMulticomponentThermo` 총 5개 템플릿에 아래 경로 삽입:
```
-I$(LIB_RGP13_SRC)/rfSpecie
-I$(LIB_RGP13_SRC)/SRKGas
-I$(LIB_RGP13_SRC)/chungTransport
-I$(LIB_RGP13_SRC)/SRKchungTakaMixture
-I$(LIB_RGP13_SRC)/include
```
- `~/.bashrc`의 `openfoam13()` 함수에 환경변수 추가:
```bash
--env LIB_RGP13_SRC="$HOME/openfoam/RGP-13/RGP-13-realFluid/src"
--env FOAM_CODE_TEMPLATES="$HOME/openfoam/RGP-13/RGP-13-realFluid/codeTemplates/dynamicCode"
```

## 6. sweep.sh — foamPostProcess 사용 불가

**증상:** `keyword type is undefined in dictionary "system/functions/functions"`  
**원인:** OF-13의 `foamPostProcess`는 thermo를 초기화하지 않는 경량 도구. coded functionObject(`type coded`)를 파싱할 수 없고, `turbulenceFields`도 난류 모델 없이 작동 불가.

**수정:** sweep.sh를 **`foamRun` 1스텝 실행 방식**으로 전면 재작성:
- chemistry off / combustionModel none 설정
- `writeObjects` functionObject로 `rho`, `mu`, `kappa`, `Cp` 출력
- controlDict/chemistryProperties/combustionProperties를 sweep 전 백업 → sweep 후 `trap EXIT`으로 자동 복원

## 8. hePsiThermo → heRhoThermo 전환

**동기:** 초임계 조건에서 의사임계점(pseudo-critical point) 근방의 급격한 밀도 구배를 정확히 포착하기 위함.

| | hePsiThermo | heRhoThermo |
|---|---|---|
| 밀도 계산 | $\rho = \psi \cdot p$ (압축성 기반) | $\rho = \rho(p, T)$ (EOS 직접 호출) |
| 적합 영역 | 고속 압축성 유동 | 저속 가변밀도, 급격한 밀도 변화 |
| 초임계 문제 | $\psi$가 의사임계점에서 급변 → 수치 불안정 | SRK의 3차 방정식 근을 직접 사용 → 정확 |

**수정:**
- `src/realFluidThermos.C` — `rhoFluidMulticomponentThermo` 인스턴스화 추가 (기존 `psiMulticomponentThermo`와 병렬)
- `codeTemplates/dynamicCode/rhoFluidMulticomponentThermo`, `rhoFluidThermo` — RGP-13 include 경로 패치
- `test/KERO/constant/physicalProperties` — `type hePsiThermo` → `type heRhoThermo`

**검증 (30bar, 300K):**

| 물성 | hePsiThermo | heRhoThermo | 차이 |
|------|-------------|-------------|------|
| rho [kg/m³] | 813.28516121**0626** | 813.28516121**0625** | ~1e-13 |
| mu [Pa.s] | 동일 | 동일 | - |
| kappa [W/m/K] | 동일 | 동일 | - |
| Cp [J/kg/K] | 동일 | 동일 | - |

아임계 조건에서 기계 정밀도 수준의 동일한 결과 확인. 초임계 의사임계점 근방에서 heRhoThermo가 더 안정적.
